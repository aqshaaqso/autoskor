GAMBARAN BESAR
--------------
AutoSkor adalah Single Page Application (SPA) frontend untuk upload dokumen
RAT koperasi, memantau antrian proses, dan menampilkan hasil penilaian kesehatan
koperasi (KSP/USP) berdasarkan Permen KUKM No. 14/2009.

Peran frontend BERUBAH dari "upload lalu tunggu & hitung skor" menjadi
"upload dokumen, beri notifikasi sukses, pantau antrian & tampilkan hasil".

Pemisahan tanggung jawab:

  Lapisan       | Tugas
  --------------|----------------------------------------------------------
  Frontend      | Upload file, notifikasi, tampilkan list & hasil skor
  Backend       | Terima file, validasi, simpan, kelola antrian & status
  Engine        | OCR, ekstrak data, hitung skor, simpan hasil

ALUR PER HALAMAN
================================================================================

HALAMAN 1 — UPLOAD (/upload)
----------------------------

  User
    │
    ├─ drag & drop banyak file (multi-file)
    │
    ├─ FE cek: total size ≤ 20MB? format OK?
    │     └─ tidak → tampilkan error
    │
    ├─ klik Upload
    │     └─ Axios POST /documents/upload
    │           └─ onUploadProgress → bar % upload
    │
    ├─ response 200
    │     └─ toast "Dokumen berhasil diupload"
    │     └─ (opsional) redirect ke /queue
    │
    └─ SELESAI — FE tidak tunggu engine


HALAMAN 2 — ANTRIAN (/queue)
----------------------------

  User buka halaman Antrian
    │
    ├─ GET /documents?status=queued
    │   GET /documents?status=processing  (opsional, bisa digabung)
    │
    ├─ Tampilkan tabel/list:
    │     nama file | tanggal upload | status
    │
    └─ Auto-refresh / tombol refresh
          (polling tiap X detik — FE hanya BACA, tidak memproses)


HALAMAN 3 — SELESAI (/processed)
--------------------------------

  User buka halaman Selesai
    │
    ├─ GET /documents?status=done
    │
    ├─ Tampilkan list dokumen yang sudah jadi
    │
    └─ User klik satu dokumen
          │
          └─ navigasi ke /processed/:id
                │
                └─ GET /documents/:id/results
                      │
                      └─ Tampilkan:
                            · ScoreSummary (skor parsial)
                            · ResultsTable (aspek dapat dihitung)
                            · NonProcessAble (aspek Manajemen)

PERBANDINGAN: ARSITEKTUR LAMA VS BARU
================================================================================

  LAMA (MVP awal)                         BARU (target)
  ─────────────────                       ─────────────────────

  [Upload 1 file]                         [Upload multi-file ≤ 20MB total]
         │                                        │
         ▼                                        ▼
  [FE tunggu proses mock]                 [FE upload → toast sukses]
         │                                        │
         ▼                                        ▼
  [FE langsung tampil skor]               [Backend → Queue → Engine]
                                                 │
                                                 ▼
                                          [FE baca status & hasil]
                                          Halaman Antrian / Selesai

  Aspek              | Lama                    | Baru
  -------------------|-------------------------|----------------------------------
  Peran FE           | Upload + proses + skor  | Upload + monitor + tampil hasil
  Loading di FE      | Ekstrak, hitung, susun  | Upload saja
  Jumlah halaman     | 1 halaman               | 3 halaman + detail
  Jumlah file        | 1 file                  | Multi file
  React Router       | Tidak dipakai           | Wajib (sidebar + routing)

================================================================================
SKOR PARSIAL & ASPEK TIDAK DAPAT DIHITUNG
================================================================================

Hasil penilaian di halaman Selesai (/processed/:id) tetap memisahkan:

  - detail[]              → aspek yang dapat dihitung dari dokumen (85 bobot)
  - tidakDapatDihitung    → aspek Manajemen (15 bobot), skor = 0
  - totalSkorParsial      → skor hanya dari aspek dapat dihitung
  - persentaseParsial     → persentase dari 85 bobot, bukan 100

Lihat: tidak bisa dihitung.txt untuk rincian komponen Manajemen.


================================================================================
ENDPOINT API (KONTRAK FE ↔ BACKEND)
================================================================================

  Endpoint                      | Method | Fungsi
  ------------------------------|--------|----------------------------------
  /documents/upload             | POST   | Upload 1+ file (multipart, ≤20MB)
  /documents?status=queued      | GET    | List dokumen dalam antrian
  /documents?status=processing  | GET    | List dokumen sedang diproses
  /documents?status=done        | GET    | List dokumen selesai
  /documents/:id/results        | GET    | Hasil skor dokumen selesai

Catatan: Path endpoint final mengikuti konvensi tim backend.
